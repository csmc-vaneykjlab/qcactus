## 💡 Key Design Decisions

### 1. Property Wrapper Pattern
All original public fields are now properties that proxy to DataModel:
```csharp
public List<double> bfs { get => _dataModel.BlankFileSizes; set => _dataModel.BlankFileSizes = value; }
```
**Why:** Maintains backward compatibility while centralizing data management

### 2. Service Initialization
Services initialized once in constructor:
```csharp
private void InitializeServices()
{
    _dataModel = new DataModel();
    _analysisEngine = new AnalysisEngine(_dataModel);
    _plottingService = new PlottingService(_dataModel, GetOutputDirectory());
    _pdfReportService = new PDFReportService(GetOutputDirectory());
}
```
**Why:** Cleaner dependency injection, single source of truth

### 3. Thin UI Layer
Form1.cs now handles only:
- User interaction (dialogs, clicks)
- UI updates (control manipulation)
- Orchestration (calling services)

**Why:** Cleaner architecture, easier to test services independently

### 4. Method Delegation Pattern
Critical methods now follow pattern:
```csharp
private void MethodName()
{
    // Keep UI logic
    var dialog = new SomeDialog();
    if (dialog.ShowDialog() == DialogResult.OK)
    {
        // Delegate processing to service
        var results = _service.ProcessData(data);

        // Display results
        UpdateUI(results);
    }
}
```
**Why:** Clear responsibility separation, easy to understand flow
